Help

Janus is a tool to federate AWS login. It uses your Guardian Google account to determine your identity and then looks up the AWS permissions you have defined in Janus' source code.

The home page shows what level of access you have for each AWS account. If you have no permissions you will need to get yourself added to Janus. Details for doing so are available in Janus' README.

Actions

For each permission you have available, you can choose between two actions.

Console

The console button will log you into AWS' web console in a session that has the selected permissions.

Developer permissions include most of Amazon's services but exclude the ability to create an IAM user with an access key/password. There is also an Account admin permission that grants full admin access to the account. These users are able to create IAM users that can manage the account without Janus.

Credentials

Clicking credentials grants you the same access as the same permission via the console, but instead of logging you in to the AWS console you will be given temporary credentials. These can be used in your application or to make API requests using the AWS CLI.

More information is available in Janus' GitHub repository.
